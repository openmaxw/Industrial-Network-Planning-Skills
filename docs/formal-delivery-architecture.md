# 正式交付件目录蓝图（强论证型）

本文件用于定义 `industrial-network-planner` 正式交付件的目标结构。

## 设计目标

正式交付件不再按方法章节组织，而按客户阅读和审批逻辑组织：

- 每章只承载一个主题
- 前后具备严格逻辑顺序
- 既给出最终方案，也保留充分的调研、评估、选择与收敛痕迹
- 图、表、清单优先，文字用于解释和收口

## 目标目录

### 1. 项目概述
- 项目背景
- 建设目标
- 建设范围
- 本次方案交付边界

### 2. 现场调研与现状结论
- 调研对象与覆盖范围
- 现网结构现状
- 现有系统与设备分布
- 已知通信关系
- 已知地址现状
- 已知机房、链路与部署条件
- 已知约束与问题

### 3. 需求与约束分析
- 业务连续性要求
- 安全隔离要求
- 运维接入要求
- 扩展与预留要求
- 抗干扰与环境适配要求
- 与既有系统兼容要求
- 施工与切换约束

### 4. 设计原则与技术路线
- 网络架构设计原则
- 分区与边界控制原则
- 地址与命名原则
- 运维与审计原则
- 实施与演进原则

### 5. 技术选择与方案比较
- 总体架构备选思路
- 分区方式备选思路
- 边界控制备选思路
- 地址规划备选思路
- 运维接入备选思路
- 最终选型及原因

### 6. 总体网络架构方案
- 总体架构说明
- 分层结构说明
- 区域定位说明
- 关键系统与边界关系说明

### 7. 网络拓扑图
- 总体拓扑图
- 重点区域拓扑图
- 主干、汇聚、接入关系说明
- 关键链路说明

### 8. 网络分区与边界控制方案
- 分区清单
- 边界清单
- 高关键对象隔离策略
- 业务接口边界策略
- 运维接入边界策略
- 访问控制矩阵

### 9. 地址、VLAN 与子网规划
- 地址规划原则
- VLAN 划分规则
- 子网组织方式
- 编号命名规则
- 既有地址继承策略
- 扩展预留策略
- VLAN、子网、IP 分配表

### 10. 关键设备部署方案
- 核心设备部署
- 汇聚设备部署
- 边界设备部署
- 运维设备部署
- 服务器与网关部署
- 部署前置条件

### 11. 通信与运维接入方案
- 业务通信组织
- 跨域访问路径
- 远程运维方式
- 审计与审批要求
- 白名单控制要求

### 12. 实施方案与切换策略
- 实施阶段划分
- 施工组织建议
- 切换策略
- 回退策略
- 验证与验收组织

### 13. 风险分析与待确认事项
- 风险项
- 待现场复核项
- 待客户确认项
- 对定版的影响说明

### 14. 实施结论与定版条件
- 已定版内容
- 定版建议
- 下发条件
- 闭环事项

### 附录
- 网络拓扑图集
- VLAN / IP / 子网明细表
- 设备部署明细表
- 访问控制矩阵明细表
- 对象与接口清单

## 图表要求

正式交付件必须优先输出：

- 总体拓扑图
- 重点区域拓扑图
- 访问控制矩阵表
- VLAN / 子网 / IP 表
- 关键设备部署表
- 定版闭环表

## 输出原则

- 文本用于解释“为什么这样设计”
- 图表用于表达“最终设计结果是什么”
- 不对客户外露 ISA95、IEC62443 等方法名称
- 但保留充分调研、评估、选择和收敛痕迹
