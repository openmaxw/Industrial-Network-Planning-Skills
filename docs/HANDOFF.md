# 项目交接说明

## 1. 项目定位

当前工作目录为 `/home/max/Industrial-Network-Planning-Skills`。

这是一个独立维护的工业网络规划项目，核心目标是：

- 组织标准化项目输入
- 进行业务、环境、网络、安全与实施分析
- 生成客户交付级正式 HTML 方案

项目应始终以“面向真实项目交付”的口径维护，而不是以历史来源、迁移背景或其他系统关系作为主叙事。

## 2. 当前结构

- `core/`：方法、规则、输入合同与文档装配规范
- `host/cli/`：命令行宿主与生成实现
- `docs/`：项目说明、架构说明与使用文档
- `output/`：本地生成结果

## 3. 当前输出要求

当前主输出为正式 HTML。

输出必须体现：

- 业务需求与系统协同
- 现场调研与设计依据
- 总体架构、安全方案与工程设计
- 冗余、性能、地址、通信与实施安排
- 风险分析与少量必要的待确认事项

## 4. 文档维护原则

- 以当前项目为中心表述
- 减少样板化、说明腔和系统中间态口吻
- 不保留容易让读者联想到旧系统来源的叙述
- 不把方案写成“数据导入后的自动拼装结果”
- 对无用、过时或误导性的迁移类说明及时清理

## 5. 后续工作建议

后续如继续优化，应优先围绕以下方向：

1. 提升正式方案的行业化表达
2. 完善业务、环境、冗余、性能与实施章节深度
3. 优化 HTML 版式与图表质量
4. 继续压缩内部工程术语在正式交付件中的暴露
