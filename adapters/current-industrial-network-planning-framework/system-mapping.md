# 当前系统映射说明

本文件用于说明当前仓库中的方法学页面、运行时数据与标准输入合同之间的映射关系。

## 1. 适用范围

仅当输入来自当前系统时使用本文件。

当前系统的主要数据入口包括：

- `formData`
- `recordCollections`
- 方法学输出页
- 导出的运行时 JSON
- 方法学案例数据

## 2. 映射原则

- 优先按信息语义映射，不按单个字段名机械映射
- 允许多个页面共同支撑同一标准输入域
- 输出页可作为设计依据的摘要来源，但不能替代原始证据

## 3. 建议映射关系

### 3.1 项目域

通常来自以下页面类别：

- 概览页
- 项目基础页
- 站点/厂区页
- 项目定位与目标页

### 3.2 调研域

通常来自以下信息：

- 调研记录
- 访谈记录
- 问题清单
- 现场观察记录
- 约束说明

### 3.3 现状网络域

通常来自以下页面类别：

- 网络拓扑
- 连接关系
- 边界/通道
- 现状结构摘要

### 3.4 资产与系统域

通常来自以下页面类别：

- 对象分配
- 设备归属
- 系统记录
- 安装位置记录

### 3.5 通信与访问域

通常来自以下内容：

- 业务流/控制流记录
- 访问路径说明
- 远程运维说明
- 边界或跨域访问记录

### 3.6 地址规划域

通常来自以下内容：

- 地址规则
- 网段记录
- 地址清单
- VLAN/子网信息

### 3.7 安全与合规域

通常来自以下内容：

- IEC62443 相关页面
- 区域与通道记录
- 边界控制记录
- 访问控制要求

### 3.8 设计输出域

通常来自以下内容：

- 设计推演页
- 结果落地页
- 输出页摘要
- 关键规则判断结果

### 3.9 假设与待确认项域

通常来自以下内容：

- 特殊判断
- 经验候选
- 缺失记录
- 未闭合问题

## 4. 系统内使用路径

建议首版按以下路径使用：

1. 在系统中完成采集、判断、推演与结果整理
2. 导出运行时 JSON
3. 按本映射文件归一到标准输入合同
4. 调用 skill 生成客户方案

## 5. 接入边界

- 当前文件仅描述映射原则，不要求首版绑定具体 UI 组件或 store 改造
- 后续若新增“标准输入包导出”功能，应以本文件为依据实现


## 6. 经验体吸收映射要求

当 skill 面向某个具体经验体工作时，除字段映射外，还必须补充以下映射：

- `navigation` 到五段式流程的映射
- `enabledRuleTopics` 到判断主题清单的映射
- `pages` 到采集页 / 判断页 / 推演页 / 输出页 / 迭代页的分类映射
- `outputs` 到客户文档章节或内部输出的用途映射

建议对每个经验体至少形成以下表：

1. 页面职责表
2. 规则主题影响表
3. 推演结论进入客户文档章节的映射表
4. 内部输出与客户输出分流表

这样 skill 才是“吸收经验体过程”，而不是“读取几个字段后写文档”。
