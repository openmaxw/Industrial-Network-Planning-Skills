# 最小输入样例

以下示例说明，若要让 `industrial-network-planner` 生成一版最小可用的客户方案，至少需要准备哪些输入。

## 1. 项目概况

- 项目名称：某新能源工厂工业网络升级项目
- 客户名称：某新能源制造企业
- 站点：华东生产基地一期
- 项目目标：完成生产网络分层分区优化、远程运维边界梳理与地址规划建议
- 项目范围：覆盖厂级核心、车间汇聚、产线接入、控制系统接入与远程维护边界
- 约束：现网需分阶段迁移，不允许长时间停机

## 2. 现状网络

- 现网采用厂级核心 + 车间交换 + 产线接入结构
- 现有控制系统、MES、视频监控与运维终端共网现象明显
- 远程运维通过统一出口接入，但边界职责不清晰

## 3. 资产与系统

- 系统：MES、SCADA、PLC、工程师站、视频监控平台、远程运维平台
- 关键对象：车间交换机、产线交换机、控制器、上位机、服务器、工业防火墙

## 4. 通信与访问

- MES 与车间系统存在跨区域访问
- 工程师站需要对 PLC 和上位机进行维护访问
- 远程厂商维护需按受控路径访问部分设备

## 5. 地址与边界

- 现有网段存在复用与划分混乱问题
- 客户希望保留部分既有地址，逐步完成规范化调整

## 6. 待确认项

- 工业防火墙品牌与部署位置待确认
- 冗余结构是否扩展到产线接入层待确认
- 远程运维平台的审计要求待确认
