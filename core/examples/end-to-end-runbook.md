# 从总经验体到客户方案的端到端运行说明

本文件用于说明 `industrial-network-planner` 如何基于 ISA95 + IEC62443 总经验体，从项目页面信息一路生成客户交付级网络方案。

它不是实现代码，而是一份执行说明，帮助后续实现者、维护者或使用者理解整个生成过程。

## 1. 目标

演示以下链路：

1. 读取总经验体中的页面信息
2. 按页面职责完成归类
3. 按标准输入模型完成归一
4. 按规则主题完成设计判断
5. 按文档装配规则生成客户方案

## 2. 输入前提

### 2.1 经验体前提

当前使用的是 ISA95 + IEC62443 总经验体。

其中：

- ISA95 主线负责结构建模与系统协同
- IEC62443 主线负责分区分域与安全边界设计

### 2.2 项目前提

假设当前项目已经在系统中完成以下工作：

- 录入企业信息、站点结构、业务流程、对象分配
- 完成主要规则主题的判断
- 完成 ISA95 协同设计和 IEC62443 安全设计推演
- 形成运行时数据导出内容

## 3. 执行步骤

### Step 1：读取经验体页面结构

读取总经验体中的页面和导航结构，识别：

- 哪些页面属于采集页
- 哪些页面属于判断页
- 哪些页面属于推演页
- 哪些页面属于输出页
- 哪些页面属于迭代页

参考文件：

- `references/experience-page-classification.md`

产出：

- 页面职责清单
- 页面读取顺序

### Step 2：提取项目事实

从采集页中提取项目事实，形成原始项目事实集。

包括：

- 项目背景与目标
- 站点结构与层级对象
- 业务流程和系统协同关系
- 对象归属和部署位置
- 现状网络、地址、连接、约束

参考文件：

- `references/experience-absorption-spec.md`
- `references/research-analysis-design-flow.md`

产出：

- 项目事实清单
- 现状对象清单
- 待确认项初稿

### Step 3：映射到标准输入模型

把页面内容按语义映射到标准输入模型。

包括：

- 项目域
- 调研域
- 现状网络域
- 资产与系统域
- 通信与访问域
- 地址规划域
- 安全与合规域
- 设计输出域
- 假设与待确认项域

参考文件：

- `references/input-contract.md`
- `references/page-to-input-mapping.md`
- `references/system-mapping.md`

产出：

- 标准输入对象

### Step 4：应用规则主题形成判断

基于启用的规则主题，对标准输入对象进行设计判断。

重点包括：

- 场景判断
- 分层判断
- 区域与通道判断
- 边界与访问控制判断
- 地址规划判断
- 冗余、稳定性、部署和实施判断

参考文件：

- `references/rule-topic-to-design-mapping.md`
- `references/design-decision-rules.md`

产出：

- 设计依据清单
- 关键设计判断
- 受约束的设计边界

### Step 5：汇总 ISA95 与 IEC62443 两条主线

将结构建模主线与安全设计主线进行汇总。

其中：

- ISA95 主线提供结构、协同、对象归属与系统关系
- IEC62443 主线提供区域、通道、边界、访问控制与风险约束

参考文件：

- `references/experience-page-classification.md`
- `references/methodology-core.md`

产出：

- 目标网络架构主结论
- 安全边界主结论
- 地址规划主结论
- 实施建议主结论

### Step 6：按文档章节装配内容

按客户交付文档结构，将事实、建议、假设和待确认项装配到对应章节中。

参考文件：

- `references/report-outline.md`
- `references/document-assembly-rules.md`
- `templates/customer-solution-template.md`

装配结果包括：

- 项目概述与建设目标
- 现状网络与调研结论
- 设计依据与方法说明
- 总体网络架构方案
- ISA95 层级建模与系统协同结构
- IEC62443 分区分域与安全边界设计
- 网络拓扑与通信路径说明
- IP 地址、VLAN 与子网规划
- 关键设备与部署建议
- 实施步骤与迁移建议
- 风险、假设与待确认项
- 附录

### Step 7：进行输出前自检

输出前检查以下问题：

- 每章是否都有可回溯来源
- 是否误把输出页当唯一依据
- 是否把建议写成事实
- 是否遗漏关键待确认项
- 是否把内部复盘内容混入客户正文

参考文件：

- `references/evidence-and-assumption-rules.md`
- `references/document-assembly-rules.md`

产出：

- 可交付的客户方案文档

## 4. 一个简化示例

### 输入侧

系统中已有以下页面内容：

- `enterprise-basic`：项目名称、客户名称、站点、目标
- `site-structure`：厂级、车间级、产线级结构
- `business-process`：MES 到产线系统的业务协同关系
- `asset-allocation`：系统、服务器、PLC、工程师站的归属
- `layer-rules`：建议采用厂级核心、车间汇聚、产线接入的三级组织
- IEC62443 边界与通道设计页：建议按运维区、DMZ、控制区划分边界并收敛远程维护路径
- `isa95-design`：形成协同结构和部署组织建议
- 安全设计页：形成分区分域和访问控制建议

### 处理后

skill 应形成：

- 项目概述：说明项目背景、范围和约束
- 总体网络架构：说明核心/汇聚/接入结构
- ISA95 建模：说明企业/站点/系统协同结构
- IEC62443 设计：说明 zone/conduit、边界和远维治理方式
- 地址规划：说明地址组织原则和预留策略
- 风险与待确认：说明品牌、部署位置、审计范围等未闭合项
