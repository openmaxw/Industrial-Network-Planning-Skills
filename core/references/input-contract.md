# 标准输入合同

## 1. 目标

标准输入合同用于描述一个工业网络规划项目的完整基础信息，使系统能够在不依赖外部上下文解释的前提下，直接生成正式方案文档。

## 2. 基本原则

- 输入直接描述项目事实与设计条件
- 字段命名保持稳定、清晰、可扩展
- 不绑定特定页面结构或临时运行时对象
- 缺失项允许保留，但应尽量减少关键章节信息断层

## 3. 重点信息域

标准输入应尽量覆盖以下信息域：

1. 项目目标
2. 范围界定
3. 业务流程
4. 设备归属
5. 安全需求
6. 环境条件
7. 实施约束
8. 设计原则
9. 架构划分
10. 技术选择
11. 网络设计
12. 工程设计
13. 网络拓扑
14. 分层设计
15. 性能设计
16. 稳定性设计
17. 冗余设计
18. 地址分段
19. 时延分析

## 4. 使用要求

输入包应能够单独被阅读和理解。

也就是说，即使脱离采集过程或整理过程，阅读输入内容的人仍能知道：

- 项目在做什么
- 为什么这样设计
- 受哪些业务、环境和工程因素约束
- 方案应如何形成
