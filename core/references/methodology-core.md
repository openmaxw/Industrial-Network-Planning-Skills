# 方法体核心执行链

本文件用于把 ISA95 + IEC62443 的方法体整理为统一执行主线。

## 1. 总体原则

本 skill 只执行既有方法体，不另起一套网络规划流程。

统一执行链为：

1. 信息采集
2. 现状理解
3. 规划判断
4. 安全设计
5. 方案成文
6. 经验迭代

## 2. 信息采集

目标：确认方案生成所需的最小必要信息。

至少应确认以下信息：

- 项目背景、目标与范围
- 站点/厂区/系统对象基础信息
- 现状网络、关键边界与主要连接
- 主要设备、系统、角色与部署位置
- 通信需求、运维需求、远程接入需求
- 现有 IP/网段/VLAN 信息
- 环境条件与实施约束
- 安全目标与边界要求

规则：

- 缺少关键信息时，不得进入确定性方案表达
- 缺失项统一转入待确认清单

## 3. 现状理解

目标：形成统一的现状认知。

应完成以下工作：

- 从 ISA95 视角识别层级、对象归属和系统协同关系
- 梳理现状网络拓扑、连接路径与主要边界
- 识别业务流、控制流、运维流和跨域访问路径
- 识别现网继承项、限制项与风险项

## 4. 规划判断

目标：把输入资料转化为网络设计判断依据。

主要判断内容：

- 分层与分区思路
- 接入与汇聚关系
- 冗余与高可用方式
- 地址规划粒度与命名方式
- 媒介、距离、干扰、安装与供电约束
- 协议承载与互联策略
- 实施可行性与迁移约束

## 5. 安全设计

目标：以 IEC62443 为主线补齐安全边界与控制要求。

主要任务：

- 划分 zone 与 conduit
- 识别边界与信任边界
- 明确访问控制原则
- 明确远程运维边界与受控路径
- 将安全需求映射到方案章节与边界说明

## 6. 方案成文

目标：把分析与设计结果组织为客户交付级方案。

要求：

- 章节稳定
- 语言专业
- 建议与事实分离
- 待确认项显式表达
- 附录清单完整

## 7. 经验迭代

目标：为后续方法体升级保留反馈入口。

应记录：

- 特殊判断
- 经验候选
- 复用建议
- 回写提示
