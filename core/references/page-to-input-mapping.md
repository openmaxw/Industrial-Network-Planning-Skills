# 页面到标准输入与客户文档章节映射

本文件用于在“总经验体页面职责归类”的基础上，进一步回答两个问题：

1. 每个页面的内容应映射到标准输入模型的哪些域
2. 每个页面的内容最终应进入客户方案的哪些章节

它是 skill 从“理解页面职责”走向“组织客户方案”的关键桥梁。

## 1. 映射原则

- 先映射到标准输入模型，再映射到客户文档章节
- 优先按页面语义职责映射，不按单一字段名机械映射
- 同一页面可以支撑多个输入域和多个文档章节
- 输出页通常作为摘要补充，不应替代采集页和推演页
- 迭代页通常进入风险、假设、待确认项或内部说明，不直接形成正文定稿结论

## 2. 标准输入模型回顾

本 skill 当前使用的标准输入模型包括：

- 项目域
- 调研域
- 现状网络域
- 资产与系统域
- 通信与访问域
- 地址规划域
- 安全与合规域
- 设计输出域
- 假设与待确认项域

## 3. ISA95 主线映射

| 页面 Key | 页面标题 | 主要映射到的标准输入域 | 最终支撑的客户文档章节 | 备注 |
|---|---|---|---|---|
| `enterprise-basic` | 企业信息 | 项目域、调研域 | 项目概述与建设目标 | 项目背景事实优先来源 |
| `site-structure` | 站点结构 | 项目域、资产与系统域、现状网络域 | 项目概述；总体网络架构方案；ISA95 层级建模与系统协同结构 | 支撑层级和站点结构认知 |
| `business-process` | 业务流程 | 调研域、资产与系统域、通信与访问域 | 现状网络与调研结论；ISA95 层级建模与系统协同结构；网络拓扑与通信路径说明 | 支撑业务协同与流程关系 |
| `asset-allocation` | 对象分配 | 资产与系统域、现状网络域 | ISA95 层级建模与系统协同结构；附录资产清单 | 是对象归属和部署角色的重要来源 |
| `scenario-rules` | 场景规则 | 设计输出域、调研域 | 设计依据与方法说明；总体网络架构方案 | 用于场景适配判断 |
| `layer-rules` | 分层规则 | 设计输出域、现状网络域 | 设计依据与方法说明；总体网络架构方案 | 用于组织核心/汇聚/接入思路 |
| `access-rules` | 接入规则 | 通信与访问域、设计输出域 | 设计依据与方法说明；网络拓扑与通信路径说明 | 用于访问和接入方式判断 |
| `technology-rules` | 技术选择规则 | 设计输出域、现状网络域 | 设计依据与方法说明；总体网络架构方案；关键设备与部署建议 | 用于技术路线判断 |
| `address-rules` | 地址规划规则 | 地址规划域、设计输出域 | 设计依据与方法说明；IP 地址、VLAN 与子网规划 | 用于地址组织和扩展判断 |
| `isa95-design` | 协同设计 | 设计输出域、资产与系统域、通信与访问域 | 总体网络架构方案；ISA95 层级建模与系统协同结构；网络拓扑与通信路径说明 | 关键推演页，优先级高 |
| `isa95-summary` | 现场概述 | 调研域、现状网络域 | 现状网络与调研结论 | 仅作为汇总摘要使用 |
| `isa95-design-result` | 设计说明 | 设计输出域 | 设计依据与方法说明；总体网络架构方案 | 仅作为摘要，不替代推演页 |
| `isa95-topology` | 协同结构 | 设计输出域、通信与访问域 | ISA95 层级建模与系统协同结构；网络拓扑与通信路径说明 | 用于表达协同结构摘要 |
| `isa95-review` | 规划复盘 | 设计输出域、假设与待确认项域 | 设计依据与方法说明；风险、假设与待确认项 | 谨慎引用，偏内部复盘 |
| `isa95-case-summary` | 案例提炼 | 设计输出域 | 通常不直接进入客户正文 | 更适合作为内部沉淀材料 |
| `special-judgements` | 特殊判断 | 假设与待确认项域、设计输出域 | 风险、假设与待确认项 | 可择要进入正文边界说明 |
| `experience-candidates` | 经验候选 | 假设与待确认项域 | 一般不进入客户正文 | 属于内部经验回写材料 |
| `iteration-summary` | 迭代总结 | 假设与待确认项域、设计输出域 | 风险、假设与待确认项 | 用于补充边界和后续建议 |

## 4. IEC62443 主线映射

当前总经验体中的 IEC62443 主线主要围绕区域、通道、边界、安全访问与风险控制展开。

| 页面类别/主题 | 主要映射到的标准输入域 | 最终支撑的客户文档章节 | 备注 |
|---|---|---|---|
| 区域识别与对象归区相关页面 | 安全与合规域、资产与系统域 | IEC62443 分区分域与安全边界设计 | 用于定义 zone |
| 通道与边界相关页面 | 通信与访问域、安全与合规域 | IEC62443 分区分域与安全边界设计；网络拓扑与通信路径说明 | 用于定义 conduit 和边界控制点 |
| 远程维护与访问控制相关页面 | 通信与访问域、安全与合规域 | IEC62443 分区分域与安全边界设计；网络拓扑与通信路径说明；实施步骤与迁移建议 | 用于说明远维与受控访问路径 |
| 威胁分析相关页面 | 安全与合规域、假设与待确认项域 | 风险、假设与待确认项；设计依据与方法说明 | 用于解释为何需要特定控制措施 |
| 风险评估相关页面 | 安全与合规域、假设与待确认项域 | 风险、假设与待确认项；实施步骤与迁移建议 | 用于明确优先级与实施收敛顺序 |
| 安全设计相关页面 | 设计输出域、安全与合规域 | IEC62443 分区分域与安全边界设计；总体网络架构方案；实施步骤与迁移建议 | 是 IEC62443 主线的关键推演结果 |
| 安全输出页/摘要页 | 设计输出域 | 设计依据与方法说明；IEC62443 分区分域与安全边界设计 | 仅为摘要补充 |
| 安全迭代页 | 假设与待确认项域 | 风险、假设与待确认项 | 不直接形成客户定稿结论 |

## 5. 客户文档章节反向来源

从客户方案章节反向看，典型来源如下：

### 5.1 项目概述与建设目标

主要来源：

- `enterprise-basic`
- `site-structure`
- 调研域中的项目目标与范围信息

### 5.2 现状网络与调研结论

主要来源：

- `enterprise-basic`
- `site-structure`
- `business-process`
- `isa95-summary`
- 现状网络相关页面

### 5.3 设计依据与方法说明

主要来源：

- `scenario-rules`
- `layer-rules`
- `access-rules`
- `technology-rules`
- `address-rules`
- IEC62443 主线中的安全规则、边界规则、风险依据
- `isa95-design-result`

### 5.4 总体网络架构方案

主要来源：

- `isa95-design`
- `layer-rules`
- `technology-rules`
- IEC62443 安全设计相关推演页

### 5.5 ISA95 层级建模与系统协同结构

主要来源：

- `site-structure`
- `business-process`
- `asset-allocation`
- `isa95-design`
- `isa95-topology`

### 5.6 IEC62443 分区分域与安全边界设计

主要来源：

- IEC62443 主线中的区域识别、通道设计、边界设计、安全设计页面
- 安全规则和威胁/风险分析页面

### 5.7 网络拓扑与通信路径说明

主要来源：

- `business-process`
- `access-rules`
- `isa95-design`
- `isa95-topology`
- IEC62443 通道、边界和远程维护设计页面

### 5.8 IP 地址、VLAN 与子网规划

主要来源：

- `address-rules`
- 地址规划相关采集页
- 目标设计中的地址规划结论

### 5.9 关键设备与部署建议

主要来源：

- `technology-rules`
- 部署规则相关页面
- IEC62443 安全设备部署建议页

### 5.10 实施步骤与迁移建议

主要来源：

- 实施约束相关采集页
- 风险评估页
- 安全设计页
- 设计推演页中的实施建议

### 5.11 风险、假设与待确认项

主要来源：

- `special-judgements`
- `iteration-summary`
- 威胁分析与风险评估页
- 各采集页中的缺失信息

## 6. 对 skill 的实际意义

有了这份映射文档，skill 才能做到：

- 不是只知道页面属于哪一类
- 还知道页面内容要进入哪个标准输入域
- 以及最终要落到客户文档的哪个章节

因此它是“页面归类文档”的下一层，也是自动生成高质量客户方案的必要条件。
