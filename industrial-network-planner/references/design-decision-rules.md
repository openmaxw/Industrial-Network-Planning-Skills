# 设计与判断规则

本文件用于约束 skill 在网络规划过程中如何做判断、如何表达结论。

## 1. 基本规则

- 先理解现状，再提出目标方案
- 先确认边界，再细化地址和设备建议
- 先识别约束，再形成实施建议
- 先标识不确定项，再生成文档结论

## 2. 优先级规则

当信息出现冲突时，优先级建议如下：

1. 明确的项目事实与现场确认结果
2. 已记录的系统化采集信息
3. 已形成的规则判断结果
4. 已有输出页摘要
5. 推导性判断

若高优先级信息缺失，不得用低优先级猜测替代事实。

## 3. 规划判断规则

### 3.1 分层与分区

- 优先依据对象角色、业务协同与边界要求确定层级与分区
- 不以单一设备名称代替结构性判断
- 跨层协同路径必须在方案中显式说明

### 3.2 冗余与高可用

- 若无明确可靠性目标，不得擅自承诺具体冗余架构
- 可给出冗余建议，但必须标记适用前提和待确认条件
- 冗余判断应结合停机容忍度、关键对象等级、物理路径条件和运维能力共同形成
- 不得默认所有区域都采用环网或双上联，应说明适用区域、规模边界与不采用原因

### 3.2A 现场环境与工程约束

- 厂房尺寸、层高、桥架路径、机柜条件、供电和接地条件应作为网络设计输入，而非实施阶段才考虑的附属信息
- 存在高温、高湿、粉尘、腐蚀、振动、冷热冲击或强电磁干扰时，应将其转写为设备等级、介质、布线、隔离与安装方式约束
- 若环境和安装条件未核实，不得输出过度确定的设备等级、安装方式和介质定版

### 3.2B 拓扑结构选择

- 星型、环型、双上联、链型等拓扑应按区域职责、对象分布、距离、可靠性目标和维护复杂度综合判断
- 环网仅在连续性要求较高、对象分散且具备明确收敛与运维边界时采用，并应说明每环规模控制思路
- 普通接入区如无明确高可用目标，不应默认采用复杂冗余结构

### 3.3 地址规划

- 优先保证清晰边界、可维护性与可扩展性
- 若现网继承限制较强，应说明继承原则与调整边界
- 若地址基础信息不完整，只能给规划原则，不能给精确分配定稿

### 3.4 设备与部署建议

- 仅在已有场景与约束支持下给出设备级建议
- 若缺少品牌、环境、安装和供电条件，不得输出过度细化的设备定稿

### 3.4A 带宽与性能

- 带宽建议应结合控制流、监控流、视频流、历史数据、远程运维流和扩容余量综合判断
- 若缺少流量、并发连接数、实时性约束或视频规模，不得将带宽值写成精确冻结值
- 广播域规模、VLAN 组织、跨区访问收敛方式和主干介质应共同纳入性能判断，而非只给链路速率结论

### 3.5 实施与迁移

- 实施建议应与停机窗口、割接条件、现网继承约束一致
- 若缺少实施窗口信息，应将实施路径标记为待确认

## 4. 安全设计规则

- zone 与 conduit 划分必须与业务对象、网络边界和访问路径一致
- 远程运维路径必须作为独立控制重点说明
- 对未确认的安全设备部署位置、策略粒度、审计范围，不得写成确定性落地结果

## 5. 成文规则

- 事实、建议、假设、待确认项必须分开表达
- 每章先结论后说明，但不能跳过依据
- 可适度归纳，不可编造依据
