# ISA95 + IEC62443 总经验体页面职责归类

本文件用于把 ISA95 + IEC62443 总经验体中的页面归类为 skill 可执行的工作结构。

它的目标不是重复页面定义，而是回答下面几个问题：

- 这个页面属于哪个阶段
- 这个页面在项目过程中承担什么职责
- 它支撑 skill 的哪一步分析或生成
- 它的内容最终应进入客户文档的哪个章节
- 它能否直接作为结论依据，还是只能作为辅助摘要

## 1. 页面类型定义

### 1.1 采集页

作用：

- 采集项目事实
- 建立对象、站点、业务、网络、约束等基础事实集

用途：

- 作为后续判断与推演的原始输入

注意：

- 采集页优先级高于输出摘要页

### 1.2 判断页

作用：

- 把采集到的事实套入规则主题进行判断
- 形成分层、分区、边界、地址、技术、安全等决策依据

用途：

- 进入“设计依据与方法说明”
- 为推演页提供规则支撑

### 1.3 推演页

作用：

- 将事实与判断结果转成结构化设计结论

用途：

- 直接支撑客户方案中的架构、安全、地址、实施章节

### 1.4 输出页

作用：

- 汇总采集、判断、推演结果
- 面向项目说明、复盘或案例提炼生成摘要

用途：

- 可作为客户文档的摘要来源
- 不能替代原始采集事实和推演链路

### 1.5 迭代页

作用：

- 记录特殊判断、经验候选和回写建议

用途：

- 进入风险、假设、边界说明或内部复盘
- 不应直接作为客户定稿结论

## 3. IEC62443 主线页面归类

以下归类基于当前 `src/methodologies/iec62443Methodology.js` 的结构与现有主题分布。

### 3.1 采集页

IEC62443 经验体中的采集页通常承担以下职责：

- 识别区域对象与边界对象
- 识别现状边界、访问关系与维护路径
- 识别威胁来源、受影响区域与关键控制点

### 3.2 判断页

IEC62443 经验体中的判断页通常来自以下规则主题：

- 场景规则
- 区域规则
- 通道规则
- 边界互联规则
- 接入规则
- 安全规则
- 部署规则
- 冗余/稳定性/性能规则（按经验体启用情况）

这些页面用于把事实转成：

- zone/conduit 判断
- 边界控制需求
- 访问控制要求
- 远程运维治理要求

### 3.3 推演页

IEC62443 的推演页通常包括：

- 区域与通道设计
- 威胁分析
- 风险评估
- 安全设计

这些页面直接支撑以下客户文档章节：

- IEC62443 分区分域与安全边界设计
- 网络拓扑与通信路径说明
- 实施步骤与迁移建议
- 风险、假设与待确认项

### 3.4 输出页

IEC62443 的输出页通常用于：

- 汇总安全设计定位
- 汇总区域、通道、边界和访问控制摘要
- 汇总风险和设计依据

这些页面可以作为客户文档的摘要来源，但不能替代：

- 原始边界对象采集
- 通道与访问路径判断
- 威胁与风险推演过程

### 3.5 迭代页

IEC62443 的迭代页作用与 ISA95 相同：

- 记录特殊判断
- 记录候选经验
- 作为后续经验沉淀依据



## 4. 两条主线如何汇总成一个客户方案

`industrial-network-planner` 面向的是一个总经验体，而不是两个彼此独立的经验体。

其中：

- ISA95 主线负责结构建模、层级组织、对象归属、业务协同和系统关系
- IEC62443 主线负责区域划分、通道组织、边界控制、远程维护、安全需求和风险约束

两条主线在客户方案中的汇总方式如下：

### 4.1 ISA95 主线主要支撑的章节

- 项目概述与建设目标
- 现状网络与调研结论
- 总体网络架构方案
- ISA95 层级建模与系统协同结构
- 部分网络拓扑与通信路径说明

### 4.2 IEC62443 主线主要支撑的章节

- 设计依据与方法说明
- IEC62443 分区分域与安全边界设计
- 网络拓扑与通信路径说明
- 实施步骤与迁移建议
- 风险、假设与待确认项

### 4.3 两条主线的汇合点

两条主线不是平行输出两份文档，而是在以下位置汇合为一个客户方案：

- 在“总体网络架构方案”中，结构建模与安全边界共同决定目标架构
- 在“网络拓扑与通信路径说明”中，业务协同路径与安全受控路径共同决定连接表达
- 在“IP 地址、VLAN 与子网规划”中，结构层级与安全边界共同影响地址组织方式
- 在“实施步骤与迁移建议”中，结构调整与安全控制上线共同影响实施路径

### 4.4 对 skill 的要求

因此 skill 在处理总经验体时，必须做到：

1. 先按 ISA95 主线建立结构认知
2. 再按 IEC62443 主线建立安全边界认知
3. 最后把两条主线汇总到一个客户交付方案中

若只读取其中一条主线，输出的方案将是不完整的。

## 5. 对 skill 的直接作用


有了这份归类文档后，skill 在读取经验体时就能按如下方式工作：

1. 先读采集页，构建项目事实
2. 再读判断页，构建设计依据
3. 再读推演页，构建设计结论
4. 再读输出页，提炼可直接用于客户文档的摘要表达
5. 最后读迭代页，把特殊判断转入风险、假设或内部说明

## 6. 使用规则

- 不能跳过采集页只读输出页
- 不能把输出页当作唯一事实来源
- 不能把迭代页中的候选建议当作客户定稿内容
- 若同一结论在采集页、判断页、推演页、输出页之间不一致，应优先回溯采集页和推演页

## 7. 后续扩展方式

后续新增经验体时，建议都补一份同类文档，至少回答：

- 页面如何归类
- 各页面支撑 skill 的哪个步骤
- 各页面如何映射到客户文档章节
- 哪些页面只用于内部，不用于客户正文
