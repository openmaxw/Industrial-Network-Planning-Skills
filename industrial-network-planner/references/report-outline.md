# 客户交付文档结构规则

本文件定义客户交付级工业网络方案的固定章节结构与写作规则。

## 1. 章节结构

建议统一采用以下主结构：

1. 项目概述与建设目标
2. 现状网络与调研结论
3. 设计依据与方法说明
4. 总体网络架构方案
5. ISA95 层级建模与系统协同结构
6. IEC62443 分区分域与安全边界设计
7. 网络拓扑与通信路径说明
8. IP 地址、VLAN 与子网规划
9. 关键设备与部署建议
10. 实施步骤与迁移建议
11. 风险、假设与待确认项
12. 结论与建议
13. 附录

## 2. 每章通用要求

每一章都应明确：

- 本章目标
- 本章结论摘要
- 本章使用的输入来源
- 基础事实记录
- 关键设计约束
- 设计响应
- 需要说明的假设或待确认项

对业务分析、调研结论、总体架构、安全设计和工程设计等核心章节，不应只保留收敛总结；必须保留足够的关键原始内容，使客户能够理解设计是如何从调研输入中推导出来的。

## 3. 重点章节说明

### 3.1 项目概述与建设目标

必须基于项目事实，不得夸大范围。

### 3.2 现状网络与调研结论

应总结当前网络状态、主要问题、关键限制与重要发现。

### 3.3 设计依据与方法说明

应说明该方案基于 ISA95 + IEC62443 方法体及项目输入形成，不应写成空泛标准罗列。

### 3.4 总体网络架构方案

应以目标结构为主，清楚说明核心网络组织方式与边界思路。

### 3.5 ISA95 层级建模与系统协同结构

应说明层级划分、对象归属和跨层协同关系。

### 3.6 IEC62443 分区分域与安全边界设计

应说明 zone、conduit、边界与访问控制原则。

### 3.7 网络拓扑与通信路径说明

应将主要连接与关键通信路径说清楚，不要求每次都画图，但要形成结构性描述。

### 3.8 IP 地址、VLAN 与子网规划

应在有足够依据时给出规划清单；信息不足时给规划原则和预留建议。

### 3.9 实施步骤与迁移建议

应强调前置条件、割接依赖、风险控制与阶段化实施。

### 3.10 风险、假设与待确认项

必须单列，不得隐藏在正文细节中。

## 4. 附录要求

附录至少包含：

- 资产清单摘要
- IP/VLAN/子网清单
- 连接关系清单
- 术语缩略语
- 待确认项清单
