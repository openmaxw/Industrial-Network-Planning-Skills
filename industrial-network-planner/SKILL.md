# Industrial Network Planner

## 1. Skill 定位

本 skill 用于把工业网络项目的调研、约束、设计规则与工程信息组织为客户交付级方案文档。

其核心任务是：

1. 理解项目目标、范围、业务流程与对象关系
2. 提炼环境条件、安全需求、工程约束与网络要求
3. 形成分层、分区、拓扑、冗余、地址、性能与实施方案
4. 输出正式、专业、可交付的方案文档

## 2. 输入要求

输入应直接描述当前项目本身的信息，而不是依赖外部运行时上下文才能理解。

应优先覆盖以下内容：

- 项目目标与范围界定
- 业务流程与设备归属
- 安全需求、安全场景、后果分析与暴露面
- 环境条件、实施约束与关键对象
- 设计原则、架构划分、技术选择与网络设计
- 工程设计、对象接入、设备选型、部署设计与连接关系
- 网络拓扑、分层设计、性能设计、稳定性设计、冗余设计、地址分段与时延分析

## 3. 输出要求

输出文档必须满足：

- 先有业务与现场，再展开设计与实施
- 保留关键事实，不只保留抽象总结
- 让客户能够看出设计依据、设计逻辑与实施边界
- 正式、简洁、专业，不带系统提示语气

默认推荐输出为正式 HTML 方案，而不是自由格式的普通 HTML 文稿。

若使用宿主程序或 CLI，应优先采用正式 HTML 输出链路，确保结果具备：

- 左侧目录导航
- 结构化章节布局
- 事实、约束、方案、待确认事项等分层展示
- 拓扑图与图表占位或 Graphviz 渲染结果

不建议绕过正式渲染链路，直接生成简化 HTML，因为那样的结果通常不具备正式交付版式。

## 5. 使用原则

- 优先服务真实项目交付
- 避免把原始调研内容压缩成失真的泛化摘要
- 对缺失信息只保留必要且有限的待确认事项
- 输出必须像正式方案正文，而不是系统生成记录
